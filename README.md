# Проект "E-commerce"

## Описание:

"E-commerce" - это электронная торговля, или электронная коммерция, проект, который будет являться ядром для интернет-магазина.

## Цель проекта  

"Создать базовую архитектуру (ядро), на основе которой в дальнейшем можно будет реализовать различные интерфейсы — от сайта до телеграм-бота.

## Архитектура и ключевые компоненты

1. Класс Product представляет товар, свойства: название, описание, цена, количество.
2. Класс Category представляет категорию товара, свойства: название, описание, товар. Атрибуты класса: category_count и product_count используются для подсчета количесва категорий и товаров.

## Установка:

1. Клонируйте репозиторий:
```
git clone https://github.com/username/myblog.git
```

2. Установите зависимости:
```
pip install -r requirements.txt
```

3. Создайте базу данных и выполните миграции:
```
python manage.py migrate
```

4. Запустите локальный сервер:
```
python manage.py runserver
```

## Тестирование
```
Этот проект использует юнит-тесты для проверки функциональности основных компонентов.
```
### Предварительные требования  
- Установленный Python 3.8 или выше  
- Установленный фреймворк тестирования (pytest)

### Запуск тестов  
Для запуска тестов выполните следующую команду:  

```bash  
pytest
```
Если вы хотите запустить только юнит-тесты:
```
pytest tests/unit/
```
Для интеграционных тестов:
```
pytest tests/integration/
```

### Ожидаемые результаты

- Тест для класса Product проверяет корректность инициализации объектов класса Product.
- Тест для класса Category проверяет корректность инициализации объектов класса Category.
- Тест test_counters проверяет подсчет количества продуктов.
- Тест test_category_count проверяет подсчет количества категорий.
- Тест test_product_price_setter проверяет работу сеттера цены.
- Тест test_product_new_product проверяет работу фабричного метода new_product.
- Тест test_category_add_product проверяет добавление продукта в категорию.
- Тест test_category_products_format проверяет формат вывода списка продуктов
- Тест test_add_wrong_product проверяет обработку попытки добавить неверный тип продукта.

После завершения тестов вы должны увидеть отчет с результатами, 
в котором указано количество успешно пройденных и проваленных тестов.
